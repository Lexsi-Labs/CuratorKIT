# Reference

Complete reference material for configuring and extending CuratorKIT. Use the
[configuration reference](configuration.md) when writing Python configs, the
[CLI & YAML reference](cli.md) when writing declarative pipelines, the
[API reference](api/index.md) when subclassing or scripting against the library,
and the [architecture page](architecture.md) when contributing.

|---|---|
| **[Configuration](configuration.md)** | Every `CuratorConfig` parameter with types and defaults |
| **[CLI & YAML](cli.md)** | `curatorkit run`, all flags, and the full YAML pipeline schema |
| **[API](api/index.md)** | Generated from the source docstrings: classes, methods, data models |
| **[Architecture](architecture.md)** | Internals: base classes, contracts, provenance model, extension points |
